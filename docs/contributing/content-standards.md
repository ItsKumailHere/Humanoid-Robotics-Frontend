# Content Standards for Physical AI & Humanoid Robotics Textbook

This document outlines the standards and conventions that should be followed when creating content for the textbook.

## Writing Style

- Use clear, concise language appropriate for the target audience
- Avoid jargon unless it's properly defined
- Maintain consistency in terminology throughout the textbook
- Write in an educational, informative tone

## Document Structure

Each chapter should follow this structure:
- Title
- Learning objectives
- Main content sections
- Summary
- Exercises or discussion questions

## Markdown Standards

- Use ATX-style headers (# Header)
- Include proper frontmatter with title, sidebar_label, and position
- Use relative links for internal navigation
- Use absolute URLs for external references
- Include alt text for images

## Code Examples

- Specify the language for syntax highlighting
- Keep examples relevant to the topic
- Include comments to explain complex code
- Test all code examples before submission

## Images and Media

- Use descriptive filenames
- Include meaningful alt text
- Compress images to optimize loading time
- Place images in the static/img/ directory
